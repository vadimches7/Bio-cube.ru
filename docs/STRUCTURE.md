# Структура проекта (где что менять)

Этот файл — “карта проекта”: где находятся блоки сайта, где лежат картинки и какие правила именования использовать.

## Главная страница

Порядок секций и их включение по режимам:

- `src/pages/Index.tsx`

## Секции (контент и верстка)

- **Команда**: `src/components/sections/TeamSection.tsx`
  - Править массив `team` (имя, роль, регалии, счётчики отзывов).
- **Отзывы (карусель)**: `src/components/sections/ReviewsSection.tsx`
  - Сейчас листаем **скриншоты** из `public/reviews/` (см. ниже).
- **Кейсы спасения (карточки)**: `src/components/sections/CasesSection.tsx`
  - Для `service` сейчас оставлен 1 реальный кейс.
- **До / После (слайдер)**: `src/components/sections/BeforeAfterSection.tsx`
  - Использует изображения из `public/cases/`.

## Картинки (быстрый доступ)

### Кейсы (до/после)

Папка:

- `public/cases/`

Ожидаемые файлы (кейс #001):

- `case-001-before.jpg`
- `case-001-after.jpg`

### Отзывы (скриншоты)

Папка:

- `public/reviews/`

Правило именования:

- `001.jpg`, `002.jpg`, … (3 цифры, без пробелов, латиница)

Ссылки в браузере:

- `/reviews/001.jpg`

Если добавили больше скринов — увеличьте `REVIEW_IMAGES_COUNT` в `src/components/sections/ReviewsSection.tsx`
или перейдём на явный список/JSON.
